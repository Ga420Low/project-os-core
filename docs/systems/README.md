# System Dossiers

Ce dossier regroupe les dossiers systeme canoniques de `Project OS`.

Un dossier systeme sert a suivre des briques externes:

- framework
- repo GitHub
- benchmark
- bridge
- produit
- stack de recherche

Il ne remplace pas une roadmap.

Resume:

- `roadmap` = ordre de construction
- `system dossier` = tri operationnel `A faire / A etudier / A rejeter`
- une `deep research` Discord/OpenClaw peut creer puis remplir ce dossier automatiquement, avant de renvoyer le `PDF` et le `.md` final sur Discord
- le dossier repo reste la version de travail, et une copie SEO-friendly est archivee en cold storage sous `archive_reports_root/deep_research/`
- convention de sortie deep research: `Markdown` repo en anglais pour la machine, `PDF` en francais pour la lecture humaine
- la profondeur de recherche est maintenant separee entre `research_profile` et `research_intensity`

## Regles

- un dossier systeme doit partir du repo reel
- il doit rester actionnable
- il doit citer des sources primaires
- il doit dire ce qu'on recupere et ce qu'on n'importe pas
- il doit expliciter la preuve attendue pour toute entree `A_FAIRE`
