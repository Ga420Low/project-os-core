# Deep Research Complex Mode

## Statut

- `active`

## But

- definir le mode `complex`
- cadrer le comite leger sans le laisser devenir un mini War Room

## Point de depart reel

- certains audits systeme ont besoin de plusieurs lanes d'evidence
- `Project OS` doit rester dans le meme contrat `go/stop` et dans la meme pile provider

## Hypothese de travail

- un parent + quelques scouts bien choisis donne un meilleur rapport profondeur/cout
- il faut parallelliser les lanes plutot que sur-raisonner dans un seul prompt

## A faire

### Light committee contract

Etat:

- `A_FAIRE`

Pourquoi il compte:

- il couvre les audits serieux de subsysteme et de stack

Ce qu'on recupere:

- parent planner
- repo scout
- official-doc scout
- GitHub scout
- optional skeptic
- parent synthesizer

Ce qu'on n'importe pas:

- swarm incontroule
- navigation arbitraire sur des sites louches

Preuves a obtenir:

- le plan d'execution `complex` liste plusieurs scout lanes
- le rapport final garde une synthese coherente et non une concatenation

Ou ca entre dans Project OS:

- `deep_research.py`
- `docs/workflow/DEEP_RESEARCH_INTENSITY_STANDARD.md`

Sources primaires:

- [Anthropic - Building effective agents](https://www.anthropic.com/engineering/building-effective-agents)
- [GitHub Docs - About forks](https://docs.github.com/pull-requests/collaborating-with-pull-requests/working-with-forks/about-forks)

## A etudier

### Skeptic lane as default

Etat:

- `A_ETUDIER`

Pourquoi il compte:

- un skeptic bien place peut casser l'hype avant la synthese finale

Ce qu'on recupere:

- contradiction pass
- angle mort check

Ce qu'on n'importe pas:

- une red-team lourde sur tous les sujets

Preuves a obtenir:

- montrer qu'un skeptic leger augmente la precision sans tripler le temps

Ou ca entre dans Project OS:

- `deep_research.py`

Sources primaires:

- [OpenAI - Graders guide](https://platform.openai.com/docs/guides/graders)
